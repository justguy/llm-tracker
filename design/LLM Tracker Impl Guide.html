LLM Tracker · Implementation Guide

Calmed terminal UI —
developer handoff.

This is the spec for rebuilding the current Bloomberg‑terminal UI as the refined Variant A2 layout. Scope: the existing Preact UI served by the hub at http://localhost:4400. Nothing in the patch protocol, MCP, or CLI surface changes — this is purely front‑end.

01Principles

what "simpler" means here
  1. Hierarchy by size + weight, not just color. The old UI leaned on amber for emphasis; we lean on type scale. Color encodes meaning (status, agent identity), not importance.
  2. Humans read the board; agents read the API. The UI optimizes for human glance‑and‑act. Power features that agents already have via MCP/CLI move into ⌘K.
  3. Keep the bracket DNA. [READ] [WHY] [EXEC] [VERIFY] stays — they're a visual signature and map cleanly to CLI verbs.
  4. One recommendation, loud. "Recommended Next" is the most visible CTA on the page. That's the answer to "what do I look at?"
  5. No modals for task detail. Expand inline; keep the board as context.

03Page layout

the refined A2 frame, top to bottom
VERTICAL STACK — 1440w reference
1. Top bar~52px. Project name + rev badge (left) · agent cluster + history cluster + ⌘K input + overflow (right). Border-bottom 1px.
2. Hero strip~140px. 3-column grid: 1.1fr · 1.2fr · 340px. Project+94% / Status grid / green Recommended Next callout. Border-bottom 1px.
3. Scratchpad~36px. Single-line truncated note row with NOTE label, text, edited timestamp, [EXPAND] [EDIT]. Expands in place on click.
4. Board header~42px. 5-col grid: 240px · 1fr · 1fr · 1fr · 1fr — SWIMLANE + P0 · Now / P1 · Next / P2 · Soon / P3 · Later.
5. Expanded laneLane label cell (240px) + 4 priority cells. One task card per cell; empty cells read — no tasks —.
6. Collapsed lanesSingle 44px row each: lane name + live dot + counts + inline progress bar + %.

04Components

the core set; everything else is composition

Bracket button

The action primitive. Inline, no pill background in default state.

// variants: default | active | soft | tone=ok|warn|block
<Bracket active>READ</Bracket>   // accent border + accent-soft fill
<Bracket>WHY</Bracket>          // 1px rule border, ink-2 text
<Bracket soft>↑</Bracket>        // no border (lane up/down)
<Bracket tone="ok">PICK</Bracket> // green CTA

Task card

3 TIERS · SCANNABLE IN 0.5S
Tier 1Mono id (T‑012) · 1 line · ink-3 · 10px; then title · Inter 600 · 14px.
Tier 2Summary. Inter 400 · 12px · ink-2 · line-height 1.5. Clamp at 4 lines.
Tier 3Meta row: ● COMPLETE claude‑opus‑4‑6 #phase1 #milestone
ActionsDashed top divider + bracket row: [READ] [WHY] [EXEC] [VERIFY]. Always visible on active card; reveal on hover for non-focus cards (optional optimization).
Status3px border-left in status color (ok / warn / ink-3 / block). The ONLY decorative color on the card.

Recommended Next callout

Green panel, top-right of hero strip. Contents:

  • Eyebrow: ★ RECOMMENDED NEXT · mono · ok · 10px · letter-spacing 1.2
  • Task id (top-right of eyebrow row)
  • Title: Inter 600 · 14px
  • One-sentence reason (priority · readiness · dependencies · approvals)
  • Two buttons: [PICK] (solid green) + [READ] (outlined)

Source: GET /api/projects/<slug>/next?limit=1. Reason string is composed client-side from the ranked-shortlist fields already returned.

Status grid

Middle hero cell. 2×3 tiles. Each tile: dot + label + big mono number. Complete & Open get the pill background; others hollow with 1px ruleSoft border. Zero-value numbers dim to ink-3.

Hero progress

5px segmented bar. Segments in order: complete (ok) · in_progress (warn) · not_started / deferred (rule-soft). Blocked contributes a 35% opacity overlay in block if present.

Command palette (⌘K)

Replaces [FILTER], [FUZZY], [COLLAPSE], [NEXT], [OVERVIEW], [SETTINGS], and most of the other 14 header actions. Single global shortcut. Command sources:

  • Projects — list from /api/projects
  • Tasks — exact filter + /fuzzy-search (deterministic) + /search (semantic)
  • Actions — next, blockers, changed, decisions, undo, redo, settings, toggle theme, help (/help)
  • Swimlanes — jump + expand/collapse all

06Interaction states

what every surface does on hover/focus/active

Task card

  • hover — border shifts from rule to ink-3; status border-left unchanged.
  • click title — expands the card inline into the task drawer (goal, deps, refs, snippets, history). Replaces the old modal.
  • drag — existing POST /move flow. Visual: card lifts by 1px shadow, dashed target highlight in destination cell.
  • bracket actions — each posts to the existing endpoint; show a 1s inline toast at the card bottom.

Lane row (collapsed)

  • hover — row gets a 1px ink-3 left accent; up/down brackets fade in on the right.
  • click — expands to full priority-column view in place.

Scratchpad

  • default — one line, truncated with ellipsis.
  • [EXPAND] — grows to full-width box; border-bottom gets stronger.
  • [EDIT] — inline textarea, cmd+enter to save (PATCH meta.scratchpad).

08Build order

ship incrementally; each step is independently deployable
  1. Tokens & typography. Drop the color + type scale into the existing Preact app. Nothing visible changes yet — but all downstream commits become diffs against a real system.
  2. Top bar + hero strip. Replaces the current first two rows. Old header brackets stay in an overflow temporarily.
  3. ⌘K command palette. Launch empty (projects + tasks only); retire [FILTER] and [FUZZY]. Add actions in subsequent PRs.
  4. Task card rework. Three-tier layout + bracket action row.
  5. Scratchpad collapse. Single-line default + expand.
  6. Inline task drawer. Replace the modal overlay for [READ] / [WHY] / [EXEC] / [VERIFY].
  7. Dark theme retune. Apply the refined palette to the existing dark mode; do not add new values.
Accessibility sweep at the end. Re-verify AA contrast for ink-2 on bg (14.5px text), and make sure the bracket buttons have hit targets ≥ 24×24 even though they look smaller. Keyboard shortcuts from the README (j / k, g g, n, p) should all work with the new layout.

09Out of scope for this pass

things to flag for a follow-up
  • Empty-state for a freshly-initialized project (npx llm-tracker init → 0 tasks).
  • Agent identity system — avatar glyph, color, tenure. (Referenced in Variant C; not in A2.)
  • Multi-project overview drawer. Current behavior preserved behind the project-name dropdown.
  • Live agent activity rail (Variant C). Consider adding as an optional right drawer in a later release.
